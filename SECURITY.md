# Security & Integrity Policy

This repository publishes the **public** version of the IAMVibeOps Manifest. It
contains no application code and no confidential material. "Security" here means
two things: **content integrity** (the published Work has not been tampered with)
and **responsible disclosure** of any concern.

## Integrity model

- Each canonical source file under `manifest/` has a recorded SHA-256 in
  [`checksums/SHA256SUMS.txt`](checksums/SHA256SUMS.txt).
- Anyone can verify the sources locally:

  ```bash
  ./verify.sh
  ```

- Continuous integration ([`.github/workflows/verify.yml`](.github/workflows/verify.yml))
  re-runs this check on every push and pull request.
- Released PDFs carry their own SHA-256 in `SHA256SUMS-release.txt`, generated by
  [`.github/workflows/release.yml`](.github/workflows/release.yml) and attached to
  each GitHub Release.
- Authorship and date provenance derive from the public, immutable Git history and
  signed tags. Optional independent timestamping (e.g. OpenTimestamps) may further
  corroborate dates.

## Reporting a concern

Please report **privately first**, do not open a public issue, if you believe:

- a published file or PDF has been altered or does not match its checksum;
- a release or tag was created that you cannot account for;
- this Work is being misrepresented, misattributed or used to imply a false
  affiliation;
- confidential material was accidentally committed to this public repository.

**How to report:** contact the author through [iamvibeops.com](https://iamvibeops.com).
Please include the affected file(s), the expected vs. observed SHA-256, and any
relevant commit, tag or URL.

## Scope

This policy covers the integrity and attribution of the published manifesto only.
It does not cover the confidential method, playbooks, prompts or product systems,
which are not part of this repository (see [NOTICE.md](NOTICE.md)).
